Navigation basics

As with everything else in Ink, navigation is easy to implement and quick to make flexible and responsive to various screen widths and if you know your way around natural language you'll quickly pick up how things work.

Tip we strongly suggest you start with the basics and read the layout section, it'll make it that much easier to get into the spirit of Ink and learn the rest.

Anatomy of a menu

A menu in Ink starts with a block-level element, such as nav or, if you prefer, and to keep things more compatible with older browsers, div

This element needs to have the class ink-navigation and will act as a wrapper to the whole thing.

Menus need to be lists, as that's what they are anyway and it's considered the industry standard best practice. So, inside your wrapper block put an unsorted list ul and list your menu items within li.

code

<nav class="ink-navigation">
   <ul class="menu">
      <li><a href="#">Item</a></li>
      <li><a href="#">Item</a></li>
   </ul>
</nav>

Submenus

Making submenus is as simple as adding a new ul inside the li you want to control the sublist and giving it the submenu class.

code

<nav class="ink-navigation">
   <ul class="menu">
      <li><a href="#">Item</a></li>
      <li><a href="#">Item with a submenu</a>
        <ul class="submenu">
            <li><a href="#">Subitem 1</a></li>
            <li><a href="#">Subitem 2</a></li>
        </ul>
      </li>
   </ul>
</nav>

Navigation styling

To style the menu, we add classes to the ul element. First, declare it a menu, using the menu class, then, decide what type of menu it's going to be: horizontal and vertical will be your primary classes, but there are others we'll describe below.

And finally, you can use utility classes for stuff like rounded corners, gradients and color.

To see code examples keep reading below for each type of menu.

Special items

Menus frequently need active and disabled items to fit most usage scenarios, so simply add the active or disabled classes to the list item you need to make either state. We strongly discourage you to use both in the same li

Available classes

  • menu Takes your list and takes care of spacing, centering and building a basic menu look. Apply to parent block element.
  • horizontal turns your list into a horizontal menu. Apply to ul
  • vertical turns your list into a vertical menu. Apply to ul
  • flat produces a flat color background, instead of the default gradient. Apply to ul
  • rounded adds rounded corners. Apply to ul
  • shadowed adds a dropshadow. Apply to ul
  • grey green blue red orange and black define the base color scheme for the menu. Apply to ul
  • active marks an item as active. Apply to li
  • disabled marks an item as disabled. Apply to li

Horizontal menu

As explained in the basics, you'll need a block element containing an unsorted list to make any menu, as long as the block has the class ink-navigation and the list has the class menu

For the horizontal variant, we'll add the horizontal tag to the list element.

code

<nav class="ink-navigation">
   <ul class="menu horizontal">
      <li><a href="#">Item</a></li>
      <li><a href="#">Item</a></li>
   </ul>
</nav>

Pagination

To create a pagination-style navigation, you need to simply replace the menu and either horizontal or vertical classes from your ul and just add the pagination class. Make sure you keep a parent element with the ink-navigation class and you're safe. Pagination-style navigation elements simply include some styling to make them more adequate for their role.

There are two special classes for navigation elements and those are the previous and next classes. Just add them to your first and last li element and make sure they contain links to navigate backward and forward in your page list.

code for a basic pagination element

<nav class="ink-navigation">
    <ul class="pagination">
        <li class="previous"><a href="#">Previous</a></li>
        <li><a href="#">1</a></li>
        <li><a href="#">2</a></li>
        <li><a href="#">3</a></li>
        <li class="next"><a href="#">Next</a></li>
    </ul>
</nav>

Pills

Pills are just another way to show your menu items as individual little rounded boxes. If you've guessed already, you probably guessed right: just put the pills class in your ul and you're set.

code for a basic pills element

<nav class="ink-navigation">
    <ul class="pills">
        <li><a href="#">Pill 1</a></li>
        <li><a href="#">Pill 2</a></li>
        <li><a href="#">Pill 3</a></li>
    </ul>
</nav>

Breadcrumbs

A breadcrumb navigation can be useful if you have several levels of depth in your site and want to let your users know where they are and easily navigate to previous items in the tree. Breadcrumbs in Ink are created by adding the breadcrumbs class to the ul element.

code for a basic breadcrumb element

<nav class="ink-navigation">
    <ul class="breadcrumbs">
        <li><a href="#">Start</a></li>
        <li><a href="#">Level 1</a></li>
        <li><a href="#">Level 2</a></li>
        <li class="active"><a href="#">Current item</a></li>
    </ul>
</nav>

Customizing the separator

If, by any chance, you'd like to use a different separator character (like *) feel free to drop this into your CSS:

.ink-navigation .breadcrumbs li:after {
   content: '*';
}

Footer menu

Footer menus are usually simple text menus with either redundant or accessory navigation. The main difference between these and the previous examples in Ink is that you can mix and match the footer class with the horizontal and vertical ones.

tip make sure your parent ink-navigation element is, itself, wrapped in a footer element.

code for an example horizontal footer menu.

<footer>
    <nav class="ink-navigation">
        <ul class="menu horizontal">
            <li><a href="#">footerItem</a></li>
            <li><a href="#">footerItem</a></li>
            ...
        </ul>
    </nav>
</footer>
